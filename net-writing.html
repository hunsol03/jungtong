이 페이지는 정보보호 네트워크 필기 정리 페이지 입니다.

TCP

  • Transmission Control Protocol의 약어, ip 프로토콜 위에서 연결형 서비스를 지원하는 전송계층 프로토콜
  • 연결형 서비스 제공
  • 전이중 방식의 양방향 가상 회선 제공
  • 신뢰성 있는 데이터 전송 보장
  • 혼잡제어, 프름제어, 오류 회복 및 재전송

UDP

  • 인터넷에서 사용하는 프로토콜 중 가장 간단한 구조
  • 비 연결형 서비스 제공
  • 헤더와 전송 데이터에 대한 체크섬 기능 제공
  • Best Effort 전달 방식 지원

ARP

  • Address Resolution Protocol의 약어
  • IP주소를 물리계층에서 이용하는 하드웨어 주소로 매핑
  • 중복된 IP할당을 찾아낸다
  • ARP 캐시가 갱신되는 경우 : 새로운 네트워크 하드웨어 추가, 새로운 IP ADDRESS가 재할당, 중복된 IP ADDRESS가 할당
  • 이웃 요청과 광고 메시지

ICMP

  • Internet Contorol Message Protocol의 약어
  • 인터넷 환경에서 오류에 관한 처리를 지원하는 용도로 사용되며, ip 패킷의 데이터 부분에 캡슐화되어 송신 호스트에게 전달
  • tcp/ip 기반의 통신망에서 전송 과정에 문제가 발생하면 라우터에 의해 icmp 메시지가 자동 발생

MTU

  • 네트워크를 통해 전송할 수 있는 최대 데이터양
  • 크기가 작으면 재전송이 자주 발생, 크기가 크면 빈 패킷이 발생하는 등 전송 효율이 저하 됨
  • 큰 MTU에서 작은 MTU로 전송시 라우터에 의해 분열

IP

  • internet Protocol의 약어, 네트워크 상에 연결된 단말기 등이 가지고 있는 고유한 주소를 IP주소라고 함
  • TTL(TIME TO LIVE) : 해당 패킷이 너무 오래 네트워크 상에 있어서 버려져야하는지의 여부를 라우터에 전송
  • TOS : QOS기능을 위한 서비스 형식 필드
  • 헤더부분 : 버전 헤더길이 ip주소 포함
